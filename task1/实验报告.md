## 1. 实验背景
本次实验旨在通过计算机视觉基础与机器学习框架的结合，完成从图像特征提取到语义分类的完整流程。
实验分为两部分：
1. 利用传统图像处理算法完成边缘检测；
2. 使用 CIFAR-10 数据集，对比传统机器学习算法与深度学习模型在图像分类任务上的表现。

* **边缘检测与预处理**：选取含有人像的彩色原始图片进行处理。由于 OpenCV 的 `cv2.imread()` 默认通道顺序为 BGR，在处理前先将其进行颜色空间转换，转化为单通道的灰度图（`cv2.CV_8U`）用于后续的梯度计算。
* **CIFAR-10 数据集**：该数据集由 60000 张 $32 \times 32$ 彩色图像组成，分为 10 个类别。其中 50000 张用于训练，10000 张用于测试。直接将此类数据输入传统算法时特征维度较高（$32 \times 32 \times 3 = 3072$ 维），容易引发维度灾难，因此需要进行相应的预处理或特征降维。

## 2. 实验过程与核心代码

### 2.1 Sobel 边缘检测
核心原理是通过二维离散微分算子求解图像的灰度近似梯度。水平（X轴）与垂直（Y轴）方向分别求导，随后对负梯度求绝对值还原至 8 位像素空间，再通过加权融合。
```python
# 核心代码
gray_img = cv2.cvtColor(img_bgr, cv2.COLOR_BGR2GRAY)
sobel_x = cv2.Sobel(gray_img, cv2.CV_64F, 1, 0, ksize=3) # 计算水平梯度
sobel_y = cv2.Sobel(gray_img, cv2.CV_64F, 0, 1, ksize=3) # 计算垂直梯度
abs_x = cv2.convertScaleAbs(sobel_x)
abs_y = cv2.convertScaleAbs(sobel_y)
sobel_combined = cv2.addWeighted(abs_x, 0.5, abs_y, 0.5, 0)
```

### 2.2 传统机器学习组：PCA降维 + SVM与随机森林
为解决高维数据直接输入 SVM 导致训练缓慢的问题，实验引入了 PCA（主成分分析）与白化处理，将特征维度降至前 150 维。
```python
# 核心代码：PCA 降维特征提取
pca = PCA(n_components=150, whiten=True, random_state=42)
X_train_pca = pca.fit_transform(X_train) # 3072 维经降维处理为 150 维

# SVM 分类器模型训练
svm_model = SVC(kernel='rbf', C=10.0, gamma='scale')
svm_model.fit(X_train_pca, y_train)
```

### 2.3 深度学习组：GPU 加速的 MLP 与 ResNet18
实验基于 PyTorch 框架构建了多层感知机（MLP，全连接层节点数设计为 1024 -> 512 -> 256）与 ResNet18 卷积神经网络。通过调用 GPU 进行硬件加速，使得全量 5 万条训练数据在可控的时间内完成了 5~10 的 Epoch 的训练迭代。
```python
# 核心代码：部署至 GPU 训练
inputs, labels = inputs.to('cuda'), labels.to('cuda')
optimizer.zero_grad()    # 梯度清零
loss = criterion(model(inputs), labels)
loss.backward()          # 误差反向传播
optimizer.step()         # 梯度下降更新参数
```

## 3. 环境配置与相关问题记录
在配置深度学习模型运行环境时，遇到了由底层环境冲突导致的运行错误，现记录如下：
* **问题描述**：`OSError: [WinError 1114] 动态链接库(DLL)初始化例程失败。Error loading c10.dll`
* **原因分析**：最初使用 Python 原生命令 `python -m venv` 创建虚拟环境，但宿主机基于 Anaconda，导致虚拟环境路径关联异常。在调用 PyTorch 底层 C++ 运算库（`c10.dll`）时，未能正确寻址到所需的 C++ 运行库（如 `msvcp140.dll`）与底层数学计算库，引发错误。
* **解决方案**：
  放弃使用原生的 venv，改用 `conda create` 重新构建完整的 Python 3.10 环境，并通过 Anaconda 渠道安装 PyTorch (`conda install pytorch ... -c pytorch`)。该方式能够自动解决并补全底层的 C++ 依赖链，最终成功配置并支持了 GPU (CUDA) 加速训练。

## 4. 实验结果对比与分析

基于配置完善的环境，实验采用 50000 张训练图像和 10000 张测试图像进行了实验，分类性能如表所述：

| 模型分类 | 算法 | 数据特征处理 | 计算平台 | 训练耗时 (s) | 测试集准确率 (%) |
| :---: | :---: | :---: | :---: | :---: | :---: |
| 传统方法 | Random Forest (200 树)| 展开为一维向量 | CPU | 13.22 | 47.41 |
| 传统方法 | SVM (RBF核, PCA150)  | 展开为一维向量 | CPU | 1020.62 | 57.15 |
| 深度学习 | MLP (全连接, 10Epoch) | 展开为一维向量 | GPU | 247.8 | 54.98 |
| 深度学习 | ResNet18 (5 Epoch)   | 保留二维几何结构| GPU | 45.3 | 72.94 |

### 结果分析：
1. **降维消耗与 SVM 复杂度**：
   实验结果显示，SVM 在特征降维至 150 维后依然达到了 57.15% 的准确度。然而，由于 SVM 的时间复杂度较高，其训练耗时超过了 1000 秒。在更大规模的数据集中，可能需要使用近似线性SVM或更严格的特征维度约束。
2. **随机森林的基准能力**：
   在 CPU 并行计算下，随机森林耗时仅约 13 秒，准确率达到 47.41%。该模型在保证了基本验证任务可用性的同时，拥有最高的训练效率。
3. **CNN (ResNet18) 的表现**：
   残差网络 ResNet18 以 72.94% 的准确率表现最佳。根本原因在于传统网络（SVM、RF和MLP）在处理数据前需要将三通道彩色图像展平为特征向量，丢失了像素间的二维空间关系和特征结构；而 ResNet 的卷积核直接在图片二维层面上滑动提取特征，保留了图形的边界、纹理以及空间连续性。此外，在此实验环境下利用 GPU 的矩阵运算优势训练不仅加速效果明显（相比大规模矩阵SVM大大减少了时间），模型的特征抽象能力也具有显著优势。

## 5. 实验总结
通过本次实验，我完成了从初步图像处理到经典机器学习、乃至深度卷积神经网络的对比构建实践，得出以下结论：
1. **输入特征与网络架构的匹配十分关键**：对于诸如图像这样具有长宽空间属性的矩阵数据，保持其原始空间结构的特征提取模式（CNN 卷积操作）远优于将其展平的基于向量特征提取的方法。网络架构需要在模型设计初始充分考虑信息特征。
2. **底层环境部署配置是基石**：记录遇到的底层 `c10.dll` 错误及其排查过程使得我意识到大型深度学习框架通常包含依赖复杂的底层计算资源（比如 C++ 标准库、数学计算核心）。遇到类似冲突时，使用像 Conda 这样全流程接管路径的包管理工具比标准原生 `venv` 更底层有效。
3. **传统模型与深度学习对特征工程的不同要求**：传统模型需要人工引入数据降维算法（如 PCA 主成分分析）提取核心约束；而深度卷积网络则更侧重于将人工干预从前置工作转化为架构设计，通过大规模算力训练在反向推导中自动捕捉所需权重特质，这种优势在处理庞大非结构化数据时较为显著。
