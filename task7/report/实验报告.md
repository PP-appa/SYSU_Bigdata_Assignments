## 三、数据集与预处理

实验使用 MovieLens `ml-latest-small` 数据集。该数据集包含 100836 条评分记录、610 个用户和 9742 部电影。实验主要使用两份文件：

- `ratings.csv`：包含 `userId`、`movieId`、`rating`、`timestamp`，表示用户对电影的评分行为。
- `movies.csv`：包含 `movieId`、`title`、`genres`，表示电影标题和电影类型标签。

本实验将评分大于等于 4.0 的记录视为用户的正反馈。为了评价推荐效果，对每个用户按时间排序，取其最后一条正反馈作为测试集，其余评分记录作为训练集。该方式相当于模拟“根据用户过去的观影记录，预测用户下一部可能喜欢的电影”。

关键代码如下：

```python
def leave_one_out_split(ratings: pd.DataFrame, min_rating: float = 4.0):
    positives = ratings[ratings["rating"] >= min_rating].copy()
    positives = positives.sort_values(["userId", "timestamp"])
    test_idx = positives.groupby("userId").tail(1).index
    test = positives.loc[test_idx, ["userId", "movieId"]].copy()
    train = ratings.drop(index=test_idx).copy()
    return train, test
```

推荐时会过滤掉用户已经评分过的电影，避免把用户已经看过的电影再次推荐给用户。

## 四、方法设计与实现

### 4.1 基于物品的协同过滤

协同过滤不直接使用电影类型或标题，而是利用用户评分行为来发现电影之间的相似性。本实验实现的是基于物品的协同过滤 ItemCF。首先构造用户-电影评分矩阵，再将矩阵转置为电影-用户矩阵，并计算电影之间的余弦相似度。

关键代码如下：

```python
user_item = train.pivot_table(
    index="userId",
    columns="movieId",
    values="rating",
    fill_value=0.0,
)
item_user = user_item.T
sim = cosine_similarity(item_user)
```

对于某个用户，系统根据其历史评分电影和候选电影之间的相似度进行加权打分：

```python
rated = ratings.to_numpy()
scores = sim @ rated
norm = np.abs(sim) @ (rated > 0)
scores = np.divide(scores, norm, out=np.zeros_like(scores), where=norm > 0)
```

其中 `sim @ rated` 表示候选电影与用户已评分电影的相似度加权和，`norm` 用于归一化，避免用户评分数量不同造成分数偏差。最后过滤已看电影，并取分数最高的 Top-K 作为推荐结果。

### 4.2 基于内容的推荐

基于内容的推荐使用电影自身特征。本实验使用 `movies.csv` 中的 `genres` 字段作为电影内容特征。首先将电影类型拆分为标签列表，再用 `MultiLabelBinarizer` 转换为多热向量。

关键代码如下：

```python
movies["genre_list"] = movies["genres"].fillna("").str.split("|")
genre_matrix = MultiLabelBinarizer().fit_transform(movies["genre_list"])
```

对每个用户，取其高评分电影的类型向量均值作为用户兴趣画像：

```python
liked_ids = user_ratings[user_ratings["rating"] >= min_rating]["movieId"]
liked_idx = [movie_to_idx[movie_id] for movie_id in liked_ids if movie_id in movie_to_idx]
profile = genre_matrix[liked_idx].mean(axis=0).reshape(1, -1)
scores = cosine_similarity(profile, genre_matrix).ravel()
```

该方法的直观含义是：如果用户过去喜欢较多动作片、科幻片，那么系统会优先推荐类型向量与该用户画像更接近的电影。相比协同过滤，内容推荐可解释性更强，但本实验只使用电影类型标签，特征相对粗糙。

### 4.3 加权混合推荐

混合推荐将协同过滤和内容推荐的候选结果进行融合。实验中先让两个模型各自产生 Top-100 候选，再对候选池进行融合重排序，最后取 Top-10 进行评价。这样可以避免只在 Top-10 内部重排导致权重变化不明显。

由于协同过滤分数和内容相似度分数含义不同、尺度不同，融合前需要按用户进行 min-max 归一化：

```python
def _minmax_by_user(df: pd.DataFrame, column: str) -> pd.Series:
    grouped = df.groupby("userId")[column]
    min_score = grouped.transform("min")
    max_score = grouped.transform("max")
    denom = (max_score - min_score).replace(0, 1.0)
    return (df[column] - min_score) / denom
```

融合公式如下：

```python
merged["cf_score"] = _minmax_by_user(merged, "cf_score")
merged["cb_score"] = _minmax_by_user(merged, "cb_score")
merged["score"] = alpha * merged["cf_score"] + (1 - alpha) * merged["cb_score"]
```

其中 `alpha` 控制两类模型的权重。当 `alpha` 较大时，混合推荐更偏向协同过滤；当 `alpha` 较小时，混合推荐更偏向内容推荐。

## 五、评价指标

本实验使用 Recall@K 评价推荐效果。由于每个用户只保留一部高评分电影作为测试目标，因此 Recall@K 表示用户被隐藏的真实喜欢电影是否出现在系统推荐的前 K 个结果中。

关键代码如下：

```python
def recall_at_k(recommendations: pd.DataFrame, test: pd.DataFrame, k: int) -> float:
    hits = 0
    total = 0
    truth = dict(zip(test["userId"], test["movieId"]))
    for user_id, target_movie in truth.items():
        user_rec = recommendations[recommendations["userId"] == user_id].head(k)
        if user_rec.empty:
            continue
        total += 1
        hits += int(target_movie in set(user_rec["movieId"]))
    return hits / total if total else 0.0
```

本实验统一使用 `Recall@10`，即每个用户推荐 10 部电影，检查测试集中隐藏的电影是否被命中。

## 七、结果分析

从实验结果看，Content-Based 的 Recall@10 为 0.0049，高于 ItemCF 的 0.0016。说明在本实验设置下，电影类型标签虽然简单，但能够较直接地反映用户历史偏好；而 ItemCF 依赖用户共同评分行为，受评分矩阵稀疏性的影响较大，电影之间的相似度估计可能不够稳定。

混合推荐的结果体现了权重参数的影响。当 `alpha = 0.3` 时，最终分数更偏向内容推荐，因此 Hybrid 的 Recall@10 接近 Content-Based；当 `alpha = 0.7` 时，最终分数更偏向协同过滤，因此 Hybrid 的 Recall@10 与 ItemCF 接近。这说明加权混合策略确实会影响推荐结果，但是否提升效果取决于两个基础模型本身的质量和权重设置。

整体 Recall@10 较低是可以解释的。本实验采用 leave-one-out 的严格评价方式：每个用户只隐藏一部喜欢的电影作为测试答案，而候选电影接近一万部，系统只能推荐前 10 部。因此即使推荐列表整体符合用户兴趣，也不一定能准确命中被隐藏的那一部电影。若改用 Recall@20、Recall@50 或引入更多内容特征，召回率可能会更高。

此外，实验中一开始直接使用两个模型各自的 Top-10 结果进行混合，改变 `alpha` 后结果几乎不变。原因是候选池过小，Hybrid 只能在少量候选中重排；同时两类分数尺度不同，直接相加会导致权重含义不清。后来改为 Top-100 候选池，并在融合前按用户归一化分数，使权重变化能够更明显地反映到最终结果中。

## 八、实验总结

本实验完成了 MovieLens 数据集上的三种推荐方法：基于物品的协同过滤、基于电影类型的内容推荐和加权混合推荐。通过实验可以看到，协同过滤主要依赖用户行为数据，内容推荐主要依赖物品特征，混合推荐则试图在两者之间取得平衡。

实验结果表明，在当前数据划分和特征设置下，基于内容的推荐效果优于 ItemCF；混合推荐的表现随 `alpha` 改变而接近不同的基础模型。后续改进方向包括引入电影标签、标题关键词或简介文本等更丰富的内容特征，尝试用户协同过滤、矩阵分解等方法，并评价 Recall@20、Recall@50 等更宽松的推荐指标。
